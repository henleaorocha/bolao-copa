# ADR-001: Dedicated Ranking Page as a Separate Route

## Status

Accepted

## Date

2026-05-27

## Context

The league panel already has navigation slots ("Ranking" items in the sidebar and bottom tab bar) that are disabled, waiting for a dedicated ranking screen. Two alternative product shapes were considered: expanding the existing panel inline, or a new route with a paginated list. The design reference (`designReferences/screens-extras.jsx`) includes a fully designed `RankingScreen` component as a separate page.

## Decision

Implement the ranking screen as a new dedicated route at `/ligas/[id]/ranking`, following the design reference layout exactly: top-3 podium → prize strip → pinned "Sua posição" card → full scrollable classification table with all league members. Activate the existing disabled "Ranking" nav slots in the sidebar and bottom tab bar.

## Alternatives Considered

### Alternative B: Expandable panel (no new route)

- **Description**: Make the "Ver tudo →" link on the existing RankingCard expand into an inline full-list view on the panel page
- **Pros**: No new route, less navigation complexity
- **Cons**: Doesn't match the design reference; breaks the tab-based navigation pattern already designed; mixing the compact dashboard view with the full ranking creates layout conflicts
- **Why rejected**: Contradicts the existing navigation architecture and design system

### Alternative C: Dedicated page with truncated list + "Ver mais" pagination

- **Description**: Same dedicated route as the chosen approach, but shows only the top 20 members by default with a load-more control
- **Pros**: Reduces initial data load for very large leagues
- **Cons**: Typical bolão leagues have 20–100 members — truncation adds friction without meaningful performance benefit; adds complexity without justification
- **Why rejected**: YAGNI — the scale of typical leagues does not warrant pagination complexity

## Consequences

### Positive

- Navigation matches the design reference, providing a consistent and expected UX
- Disabled nav slots are activated, completing the league navigation structure
- Full member list is immediately visible without extra interaction

### Negative

- Requires a new API endpoint returning all league members (vs. the current top-5 limit) with the correct tiebreaker logic applied server-side

### Risks

- If a league grows very large (500+ members), an all-at-once list may feel long. This can be addressed in a future phase with virtual scrolling if needed.
