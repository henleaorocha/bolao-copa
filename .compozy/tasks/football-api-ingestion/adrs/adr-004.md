# ADR-004: Manual result entry locks a match from automatic overwrite

## Status

Accepted

## Date

2026-06-01

## Context

Automatic ingestion runs on a fixed hourly schedule, every day (the same mechanism used today). Because the free source can be late or wrong during the live tournament, a global operator can enter or correct a match result. Without a precedence rule, the next hourly run could revert the operator's correction back to the (stale/wrong) upstream value, causing scores to flip back and forth.

## Decision

A match that has received a **manual result entry** is **protected from automatic overwrite**: subsequent automatic runs skip updating that match's score/status. A **global operator can still change it** at any time (including re-entering a value or, if desired, releasing it back to automatic control). Automatic ingestion remains the default for every match that has not been manually touched.

Operator authority is limited to two named accounts: `hen.leao.rocha@gmail.com` and `henrique.rocha@arkmeds.com`. Operator result control is reached via a **fixed, unlisted URL** the operator types directly — there is no link from the participant UI and no change to the current interface.

## Alternatives Considered

### Alternative 1: Last-write-wins (automatic always overwrites)

- **Description**: Every hourly run overwrites all matches from the source.
- **Pros**: Simplest; source is always canonical.
- **Cons**: Destroys operator corrections; scores can oscillate when the source is wrong/late.
- **Why rejected**: Defeats the purpose of having a timely manual override.

### Alternative 2: Manual entry permanently freezes the match (no later operator edits)

- **Description**: Once entered manually, a match is immutable.
- **Pros**: Maximum stability.
- **Cons**: Operator cannot fix a mistaken manual entry.
- **Why rejected**: Too rigid; operators need to correct their own corrections.

## Consequences

### Positive

- Operator corrections are durable; no oscillation between manual and automatic values.
- Timeliness during the live tournament is guaranteed regardless of upstream lag.

### Negative

- Introduces a per-match "manually controlled" distinction that must be tracked and surfaced to operators so they know which matches automation no longer touches.

### Risks

- An operator manually sets a match early and forgets it, so a later official correction from the source is ignored → mitigation: operators can release a match back to automatic control; Phase 2 results screen highlights manually-controlled matches.
