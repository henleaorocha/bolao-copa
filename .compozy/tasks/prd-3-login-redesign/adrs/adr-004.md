# ADR-004: Brand Colors — Inline Styles

**Status:** Accepted  
**Date:** 2026-05-23

## Decision

Use `style={{ }}` inline prop for all brand-specific hex colors in the login page component (`#244C5A`, `#FFC72C`, `#0097A9`). Tailwind v4 utility classes cover layout, spacing, and typography; inline styles carry only the three opinionated brand colors and opacity-derived variants (e.g., `rgba(255,255,255,0.08)`).

## Alternatives Considered

**Register as Tailwind v4 theme tokens in globals.css:** Add `@theme { --color-brand-dark: #244C5A; --color-brand-primary: #FFC72C; --color-brand-secondary: #0097A9; }` and use `bg-brand-dark` etc. Rejected for this PRD: adds a globals.css change with potential side effects on other pages, and the token extraction belongs in a future design system PRD rather than a single-screen visual fix.

## Consequences

- `globals.css` remains unchanged — zero risk of breaking existing pages.
- The three brand color values are co-located in `app/login/page.tsx`; if they change, only that file needs updating.
- Future design system PRD will extract these into proper tokens at that time.
