# ADR-002: Icon Library — lucide-react

**Status:** Accepted  
**Date:** 2026-05-23

## Decision

Install `lucide-react` as the icon library for the login page redesign. The login page requires three icons: Trophy, Flame, and Lock. `lucide-react` provides all three as named exports, is tree-shakeable (no bundle bloat from unused icons), and integrates natively with React/TypeScript via typed props (`size`, `strokeWidth`, `className`, `style`).

## Alternatives Considered

**Inline SVG paths:** Hardcode the 3-4 SVG `<path>` strings directly in the login page component. Rejected because SVG paths for non-trivial icons (trophy) are fragile to maintain and duplicated if icons appear on future screens.

**Extract Icon component from designReferences/icons.jsx:** Port the prototype's window-scoped `Icon()` function into a proper local component. Rejected because it replicates work already done by `lucide-react`, adds a file to maintain, and the prototype paths are not identical to the target design (it uses Lucide icon names internally anyway).

## Consequences

- One new production dependency (`lucide-react`). Package is well-maintained (MIT license, 10k+ GitHub stars, Lucide project).
- All future screens in this project can import Lucide icons consistently.
- Tree-shaking ensures only the three used icons are bundled for the login page.
