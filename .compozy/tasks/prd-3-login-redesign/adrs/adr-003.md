# ADR-003: LoginButton Styling — In-place Replacement

**Status:** Accepted  
**Date:** 2026-05-23

## Decision

Update `components/LoginButton.tsx` in-place to replace existing Tailwind classes with the new design (white background, slate-900 text, rounded-2xl, scale hover/active animations, updated label text "Continuar com Google"). The OAuth logic (`signInWithOAuth` call, `redirectTo` construction) is untouched.

## Alternatives Considered

**New styled wrapper at page level:** Keep `LoginButton` as a logic-only component (no `className`) and apply styles from the login page via a wrapper element. Rejected because it introduces indirection for zero benefit — this button has no other callers in the current codebase.

**Inline the button in the login page:** Move the `signInWithOAuth` logic directly into `app/login/page.tsx` as a client island and delete `LoginButton.tsx`. Rejected because it reduces the page to having a client boundary only for a button, complicating the component tree, and removes a reusable component unnecessarily.

## Consequences

- Single file to change for button styling: `components/LoginButton.tsx`.
- No new component files created.
- If a second page ever needs the Google button, it can import `LoginButton` directly with the new styles already applied.
