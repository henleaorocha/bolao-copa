# ADR-001: League Hub as Full /ligas Page Redesign

## Status

Accepted

## Date

2026-05-23

## Context

After login, users need a single, purposeful landing screen where they can see all their available leagues and choose where to play. The current `/ligas` page uses a two-tab design ("Minhas Ligas" + "Descobrir") that fragments the experience. The product requires a unified card grid showing all public leagues plus the user's private leagues, with a visual "Criar nova liga" entry point and a Copa countdown banner. Three structural approaches were evaluated.

## Decision

Replace the existing `/ligas` page with a full redesign implementing the new league hub layout. The post-login redirect is updated from `/dashboard` to `/ligas`. A unified data view combines public leagues and user private memberships into a single card grid.

## Alternatives Considered

### Alternative B: New route `/home` as post-login landing

- **Description**: Create `app/home/page.tsx` as the league selection hub; leave `/ligas` unchanged.
- **Pros**: Zero risk of breaking existing `/ligas` flows.
- **Cons**: Duplicates league-fetching logic; introduces a second URL for league discovery, causing navigation confusion.
- **Why rejected**: Route proliferation and duplicated data-fetching outweigh the stability benefit.

### Alternative C: Incremental tab adaptation

- **Description**: Keep the two-tab structure but reskin it to match the new design reference.
- **Pros**: Lowest-risk change — visual only.
- **Cons**: The new design has no tabs; forcing them in creates UX inconsistency and does not match the product vision.
- **Why rejected**: Does not deliver the unified, single-view experience required by the design reference.

## Consequences

### Positive

- Single canonical route (`/ligas`) for all league-related entry points.
- Users land directly on league selection after every login — zero extra clicks to reach their league.
- Unified data model: one view covers all league types without tab-switching friction.

### Negative

- The current two-tab `/ligas` design is removed; any deep-linked behavior relying on the "Descobrir" tab will break.

### Risks

- Users who bookmarked `/ligas` expecting the old tab layout will see the new design — low risk since the feature is early-stage.
- The `/dashboard` redirect from auth callback must be updated carefully to avoid auth loop regressions.

## Implementation Notes

- Update the post-login redirect target in the auth callback from `/dashboard` to `/ligas`.
- The unified data call must merge: (a) all `access_type='open'` leagues, and (b) user's private memberships — deduplicating where the user is also a member of a public league.
- "Test Bolao" (the universal public league) is identified by a hardcoded slug or a `is_main` flag and pinned as the first card with a "PRINCIPAL" badge.
- "Criar nova liga" card is rendered as a static visual element with no action wired up in this phase.
