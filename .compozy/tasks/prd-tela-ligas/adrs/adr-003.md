# ADR-003: Test Bolao Identification via MAIN_LEAGUE_ID Env Var

## Status

Accepted

## Date

2026-05-23

## Context

The "Test Bolao" league must always appear first in the Leagues Hub with a "PRINCIPAL" badge, regardless of any future name or slug changes. A reliable, zero-migration strategy is needed to identify it at the data layer. Three options were evaluated.

## Decision

Use a `MAIN_LEAGUE_ID` environment variable (added to `.env.local` and production secrets) containing the UUID of the "Test Bolao" league. The `getLeaguesHub()` function reads this value and marks the matching league as `is_main: true`. No database schema migration required.

## Alternatives Considered

### Alternative: Hardcoded name comparison

- **Description**: Compare `league.name === 'Test Bolao'` (or a `MAIN_LEAGUE_NAME` env var) inside the query or app code.
- **Pros**: No env secret needed; self-documenting.
- **Cons**: Name changes break the identification silently; string comparison is fragile.
- **Why rejected**: The UUID is stable and immutable; a name is not.

### Alternative: `is_main` boolean column in `leagues` table

- **Description**: Add `is_main BOOLEAN DEFAULT false` to the `leagues` table and set it to `true` for Test Bolao.
- **Pros**: Fully data-driven; no code changes if the main league ever changes.
- **Cons**: Requires a DB migration and an RLS policy update; over-engineered for a single fixed league.
- **Why rejected**: The env var approach satisfies the requirement with zero DB changes; the `is_main` column can be added in a future phase if multiple "main" leagues are ever needed.

## Consequences

### Positive

- Zero database migration needed.
- The UUID never changes, so identification is stable across name changes.
- Simple to implement: one `process.env.MAIN_LEAGUE_ID` read in `getLeaguesHub()`.

### Negative

- The UUID must be kept in sync between all environments (local, staging, production).
- If the Test Bolao league is ever re-created with a new UUID, the env var must be updated and redeployed.

### Risks

- If `MAIN_LEAGUE_ID` is not set, the hub still renders — Test Bolao simply appears in the normal sort order without the PRINCIPAL badge. This is a graceful degradation, not a crash.

## Implementation Notes

- Add `MAIN_LEAGUE_ID=<uuid>` to `.env.local` and production environment.
- In `getLeaguesHub()`: after fetching leagues, set `is_main = (league.id === process.env.MAIN_LEAGUE_ID)`.
- In the sort order: Test Bolao (is_main) is pinned to position 0 before all other ordering logic.
