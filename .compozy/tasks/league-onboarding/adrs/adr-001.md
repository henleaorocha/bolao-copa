# ADR-001: Per-League Welcome Onboarding Flow

## Status

Accepted

## Date

2026-05-23

## Context

The bolão app needs a welcome onboarding modal sequence to orient users when they first enter a league — whether they created it or joined via invite. Three strategies were considered for when and how many times the 4-screen flow (How it works → Time rules → Scoring → Invite) should be shown.

Key constraints:
- The codebase has no first-visit tracking today; `league_members.joined_at` exists but is unused for onboarding logic
- Users can belong to multiple leagues simultaneously
- Screen 4 (Invite) serves a distinct purpose for creators (recruiting members) vs. joiners (spreading the network)
- The flow must be skippable only by completing it (no explicit skip button, matching design references)

## Decision

Show the complete 4-screen welcome flow every time a user accesses a **new league for the first time** — both after creating a league and on first access after joining via invite. Completion is tracked by a `onboarded_at` timestamp column on `league_members`. Screen 4 shows context-aware text: creators see "Sua liga foi criada!"; joiners see "Você entrou em [League Name]!". After completing the flow the user lands on the league's main page.

## Alternatives Considered

### Alternative B: Global first + Screen 4 per league

- **Description**: Show Screens 1–3 only once globally (first-ever league). For all subsequent leagues show only Screen 4.
- **Pros**: Less repetition for power users; simpler screen 1–3 tracking (single user flag)
- **Cons**: Requires two distinct tracking flags (global + per-league); screens 1–3 feel tied to the first league context, not each new league
- **Why rejected**: Added complexity for a small edge case (users with many leagues); and the educational screens have value in every league context since rules and scoring reminders help engagement

### Alternative C: Full flow only for creators, joiners see Screen 4 only

- **Description**: League creators always see all 4 screens. Joiners via invite see only Screen 4 (Invite) with a welcome message.
- **Pros**: Less friction for joiners; Screen 4 invite still drives network growth
- **Cons**: Joiners miss educational context (scoring, deadlines); inconsistent experience between user types
- **Why rejected**: Educational screens increase engagement and reduce support questions about rules — both creator and joiner benefit from seeing them

## Consequences

### Positive

- Consistent, predictable onboarding experience for all users in all leagues
- `onboarded_at` timestamp provides useful analytics (time-to-complete, drop-off by step)
- Each league context gets its own first-impression moment, reinforcing that leagues are independent experiences
- Invite step at the end directly drives member growth at the moment of highest motivation

### Negative

- Users in many leagues will see Screens 1–3 repeatedly (content is largely the same)
- Requires a database migration to add `onboarded_at` to `league_members`

### Risks

- If `onboarded_at` flag is not set atomically (e.g., user closes browser on Screen 3), the flow re-triggers on next visit — mitigated by setting the flag on Screen 1 completion or on modal open
