# Initialize Next.js 13+ project with TypeScript, Tailwind CSS, and Supabase client

## Overview

This task establishes the foundational Next.js 13+ App Router project structure with TypeScript, Tailwind CSS styling, and Supabase client integration. No dependencies—this is the entry point for the entire Foundation phase. A successful setup enables all downstream tasks (middleware, API routes, pages) to build on a stable base.

<critical>
Read the **TechSpec "System Architecture"** section to understand the Next.js + Supabase + middleware pattern. The App Router is mandatory (not Pages Router). TypeScript strict mode must be enabled. All code—variable names, comments, logs—must be in Portuguese (PT-BR) per PRD constraint.
</critical>

<requirements>

1. MUST initialize a new Next.js 13+ project using `create-next-app@latest` with:
   - App Router (not Pages Router)
   - TypeScript enabled
   - Tailwind CSS enabled
   - ESLint configured
   - No example code (clean slate)

2. MUST install required dependencies:
   - `@supabase/ssr` (pacote oficial atual; `@supabase/auth-helpers-nextjs` está deprecated desde 2024)
   - `@supabase/supabase-js` (Supabase client)
   - `typescript` and `@types/*` for development

3. MUST create `.env.local` template with Supabase credentials:
   - `NEXT_PUBLIC_SUPABASE_URL` (public)
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` (public)
   - `SUPABASE_SERVICE_ROLE_KEY` (secret, optional for Foundation)
   - `SUPABASE_DB_PASSWORD` (secret; necessário para `supabase db push` via CLI local)

4. MUST configure `tsconfig.json` with:
   - `strict: true`
   - `noUnusedLocals: true`
   - `noUnusedParameters: true`
   - Path aliases for clean imports (e.g., `@/lib/*`)

5. MUST create a helper file `lib/supabase/client.ts` for Supabase server client initialization (used by middleware and API routes)

6. SHOULD initialize `.gitignore` to exclude `.env.local`, `node_modules/`, `.next/`

</requirements>
