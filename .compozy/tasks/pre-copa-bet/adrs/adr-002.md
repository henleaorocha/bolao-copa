# ADR-002: Bet Status Delivered via Extended LeagueDetail Response

## Status

Accepted

## Date

2026-05-24

## Context

The league detail page (`app/ligas/[id]/page.tsx`) must know whether the current user has already placed their pre-Copa champion/vice bet in order to decide whether to show the bet modal. Two strategies were evaluated: extending the existing `GET /api/leagues/{id}` response, or making a separate API call on mount.

## Decision

Add `has_champion_bet: boolean` to the `LeagueDetail` type and compute it in the existing `GET /api/leagues/{id}` route handler by querying the `champion_bets` table. The page already awaits this response before rendering, so no additional waterfall is introduced.

## Alternatives Considered

### Alternative 1: Separate `GET /api/leagues/{id}/champion-bet` call

- **Description**: Page makes a second fetch after mounting to check whether a bet exists.
- **Pros**: League endpoint stays unchanged; bet data is cleanly separated.
- **Cons**: Waterfall request on every league visit. Two round trips before the modal decision is made. Adds flicker risk.
- **Why rejected**: Adds latency on every league page load. The pattern is inconsistent with how `user_onboarded_at` is already delivered inline.

## Consequences

### Positive

- Single request determines both welcome modal and bet modal state.
- Consistent with existing `user_onboarded_at` pattern.
- No new client-side fetch logic needed.

### Negative

- Slightly increases complexity of the `GET /api/leagues/{id}` handler (one extra DB query or JOIN).
- `LeagueDetail` type grows by one field.

### Risks

- If the `champion_bets` query fails, the entire league response could fail. Mitigation: use a LEFT JOIN / `EXISTS` subquery rather than a separate query; absorb any error by defaulting to `has_champion_bet: false`.

## Implementation Notes

In the route handler, add a `SELECT EXISTS(SELECT 1 FROM champion_bets WHERE user_id = $user AND league_id = $league)` subquery (or Supabase `.maybeSingle()` pattern) alongside the existing league fetch.
